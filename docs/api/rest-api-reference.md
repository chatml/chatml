# REST API Reference

ChatML's Go backend exposes a REST API on port 9876 for all frontend-to-backend communication. This document covers every endpoint, organized by resource. All requests use JSON bodies and responses.

## Base URL

```
http://localhost:9876
```

## Authentication

All requests (except `/health`) require a Bearer token:

```
Authorization: Bearer <auth-token>
```

The token is generated by the Tauri shell on startup and shared with the frontend.

## Rate Limits

| Scope | Limit | Applies To |
|-------|-------|------------|
| Agent spawns | 10/min per IP | `POST /api/repos/{id}/agents` |
| Conversations | 20/min per IP | `POST .../conversations`, `POST .../summary` |
| Messages | 60/min per IP | `POST .../message`, `POST .../messages` |
| Comments | 60/min per IP | `POST .../comments` |
| Search | 20/min per IP | `GET .../issues/search` |

Rate limit responses return HTTP 429 with a `Retry-After` header.

## Authentication

### `POST /api/auth/github/callback`

Exchange a GitHub OAuth authorization code for an access token.

**Request:**
```json
{ "code": "abc123" }
```

**Response:** `200 OK` on success, stores token in Stronghold vault.

### `POST /api/auth/token`

Set a GitHub access token directly (for manual configuration).

**Request:**
```json
{ "token": "ghp_xxx" }
```

### `GET /api/auth/status`

Check whether GitHub authentication is configured.

**Response:**
```json
{
  "authenticated": true,
  "username": "octocat",
  "avatarUrl": "https://..."
}
```

### `POST /api/auth/logout`

Clear GitHub credentials from Stronghold vault.

### `POST /api/auth/linear/callback`

Exchange a Linear OAuth authorization code for an access token.

**Request:**
```json
{ "code": "abc123" }
```

### `GET /api/auth/linear/status`

Check whether Linear authentication is configured.

**Response:**
```json
{ "authenticated": true }
```

### `POST /api/auth/linear/logout`

Clear Linear credentials.

## Dashboard

### `GET /api/dashboard/data`

Fetch all workspaces, sessions, and conversations in a single request. Used on app startup to populate the sidebar.

**Response:**
```json
{
  "repos": [...],
  "sessions": [...],
  "conversations": [...]
}
```

### `GET /api/prs`

List all open pull requests across all workspaces.

**Response:** Array of PR objects with status, checks, and review information.

### `POST /api/resolve-pr`

Resolve a GitHub PR URL to structured data for the "create session from PR" flow.

**Request:**
```json
{ "url": "https://github.com/org/repo/pull/42" }
```

**Response:**
```json
{
  "owner": "org",
  "repo": "repo",
  "number": 42,
  "title": "Add feature X",
  "branch": "feature/x",
  "baseBranch": "main"
}
```

### `GET /api/avatars`

Look up GitHub avatar URLs for usernames.

**Query Parameters:**
| Parameter | Description |
|-----------|-------------|
| `usernames` | Comma-separated GitHub usernames |

## Workspaces (Repos)

### `GET /api/repos`

List all registered workspaces.

**Response:**
```json
[
  {
    "id": "repo_abc",
    "name": "my-project",
    "path": "/Users/me/code/my-project",
    "branch": "main",
    "remote": "origin",
    "branchPrefix": "github",
    "customPrefix": "",
    "createdAt": "2025-01-15T10:00:00Z"
  }
]
```

### `POST /api/repos`

Register a new workspace.

**Request:**
```json
{
  "path": "/Users/me/code/my-project"
}
```

The backend detects the git repository, reads the default branch and remote, and creates the record.

### `GET /api/repos/{id}`

Get a single workspace by ID.

### `GET /api/repos/{id}/details`

Get extended workspace details including remote URLs and branch information.

### `PATCH /api/repos/{id}`

Update workspace settings.

**Request:**
```json
{
  "branchPrefix": "custom",
  "customPrefix": "dev/"
}
```

### `DELETE /api/repos/{id}`

Remove a workspace registration (does not delete the repository on disk).

### `GET /api/repos/{id}/remotes`

List git remotes for the workspace.

### `GET /api/repos/{id}/branches`

List all branches in the workspace.

### `POST /api/repos/{id}/branches/analyze-cleanup`

Analyze branches for potential cleanup (merged, stale, orphaned).

### `POST /api/repos/{id}/branches/cleanup`

Execute branch cleanup based on analysis results.

**Request:**
```json
{
  "branches": ["feature/old-branch", "fix/resolved"],
  "deletionMode": "local"
}
```

## Sessions

### `GET /api/repos/{id}/sessions`

List all sessions for a workspace.

**Response:** Array of session objects with status, priority, task status, PR info, and stats.

### `POST /api/repos/{id}/sessions`

Create a new session. This creates a git worktree and branch.

**Request:**
```json
{
  "name": "Add user authentication",
  "branch": "feature/auth",
  "task": "Implement OAuth login flow",
  "targetBranch": "origin/develop"
}
```

**Response:**
```json
{
  "id": "sess_abc",
  "workspaceId": "repo_abc",
  "name": "Add user authentication",
  "branch": "feature/auth",
  "worktreePath": "/Users/me/.chatml/workspaces/my-project/add-user-authentication",
  "status": "idle",
  "priority": 0,
  "taskStatus": "backlog",
  "createdAt": "2025-01-15T10:30:00Z"
}
```

### `GET /api/repos/{id}/sessions/{sessionId}`

Get a single session by ID.

### `PATCH /api/repos/{id}/sessions/{sessionId}`

Update session properties.

**Request:**
```json
{
  "name": "New name",
  "priority": 2,
  "taskStatus": "in_progress",
  "pinned": true,
  "archived": false
}
```

### `DELETE /api/repos/{id}/sessions/{sessionId}`

Delete a session. Stops any running agents, removes the worktree, and deletes the branch.

### `GET /api/repos/{id}/sessions/{sessionId}/changes`

Get file changes (additions/deletions) for the session vs its base branch.

### `GET /api/repos/{id}/sessions/{sessionId}/branch-commits`

List commits on the session's branch since it diverged from the base branch.

### `GET /api/repos/{id}/sessions/{sessionId}/git-status`

Get `git status` for the session's worktree.

## Session PR & Branch

### `GET /api/repos/{id}/sessions/{sessionId}/pr-status`

Get the pull request status for a session.

**Response:**
```json
{
  "prStatus": "open",
  "prUrl": "https://github.com/org/repo/pull/42",
  "prNumber": 42,
  "hasMergeConflict": false,
  "hasCheckFailures": false
}
```

### `GET /api/repos/{id}/sessions/{sessionId}/pr/generate`

Generate a PR description using AI based on session changes and conversation context.

### `POST /api/repos/{id}/sessions/{sessionId}/pr/create`

Create a pull request on GitHub.

**Request:**
```json
{
  "title": "Add user authentication",
  "body": "## Summary\n...",
  "baseBranch": "main",
  "draft": false
}
```

### `GET /api/repos/{id}/sessions/{sessionId}/branch-sync`

Check branch sync status (ahead/behind base branch, conflicts).

**Response:**
```json
{
  "status": "behind",
  "aheadBy": 3,
  "behindBy": 5,
  "hasConflicts": false,
  "conflictFiles": []
}
```

### `POST /api/repos/{id}/sessions/{sessionId}/branch-sync`

Sync session branch with base branch (rebase or merge).

**Request:**
```json
{
  "strategy": "rebase"
}
```

### `POST /api/repos/{id}/sessions/{sessionId}/branch-sync/abort`

Abort an in-progress sync operation (rebase or merge).

## Conversations

### `GET /api/repos/{id}/sessions/{sessionId}/conversations`

List all conversations in a session.

### `POST /api/repos/{id}/sessions/{sessionId}/conversations` (rate limited)

Create a new conversation and optionally send the first message.

**Request:**
```json
{
  "type": "task",
  "message": "Implement the login form component",
  "model": "claude-sonnet-4-20250514",
  "toolPreset": "full",
  "enableCheckpointing": true,
  "maxBudgetUsd": 5.0,
  "maxTurns": 50,
  "maxThinkingTokens": 50000,
  "resumeSessionId": null,
  "forkSession": false,
  "linearIssue": "CHA-42"
}
```

### `GET /api/conversations/{convId}`

Get a single conversation by ID (top-level endpoint for direct access).

### `GET /api/conversations/{convId}/messages`

Get all messages in a conversation.

**Response:** Array of message objects ordered by position, including tool usage, thinking content, and timeline entries.

### `POST /api/conversations/{convId}/messages` (rate limited)

Send a message in an existing conversation.

**Request:**
```json
{
  "content": "Now add form validation",
  "attachments": []
}
```

### `POST /api/conversations/{convId}/stop`

Stop the running agent for a conversation.

### `GET /api/conversations/{convId}/streaming-snapshot`

Get the latest streaming snapshot for recovering state after a reconnection.

### `GET /api/conversations/{convId}/drop-stats`

Get message drop statistics (for diagnosing streaming issues).

### `POST /api/conversations/{convId}/rewind`

Rewind files to a checkpoint.

**Request:**
```json
{ "checkpointUuid": "chk_01ABC123" }
```

### `POST /api/conversations/{convId}/plan-mode`

Toggle plan mode for a conversation.

**Request:**
```json
{ "enabled": true }
```

### `POST /api/conversations/{convId}/approve-plan`

Approve a plan proposed by the agent.

### `POST /api/conversations/{convId}/answer-question`

Answer a question asked by the agent (via the AskUserQuestion hook).

**Request:**
```json
{ "answer": "Use JWT tokens for authentication" }
```

### `DELETE /api/conversations/{convId}`

Delete a conversation and its messages.

### `GET /api/conversations/active-streaming`

List all conversations currently streaming (across all sessions).

## Error Responses

All endpoints return standard HTTP status codes:

| Status | Meaning |
|--------|---------|
| 200 | Success |
| 201 | Created |
| 400 | Bad request (invalid parameters) |
| 401 | Unauthorized (missing or invalid token) |
| 404 | Resource not found |
| 429 | Rate limit exceeded |
| 500 | Internal server error |

Error responses include a message:
```json
{ "error": "session not found" }
```
